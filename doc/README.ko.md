Astro Studio는 AI 이미지 제작을 위해 설계된 데스크톱 플랫폼입니다. 특정 모델 프로바인더에 종속되지 않고, 서드파티 이미지 생성 API를 자유롭게 연결하여 다양한 서비스와 모델, 지속적인 창작 활동을 한 곳에서 관리할 수 있도록 합니다.

단일 생성 요청을 해결하는 것을 넘어, 이미지 생성을 안정적이고 오래 사용할 수 있는 장기적인 창작 워크플로우로 전환하는 것을 목표로 합니다.

## 제품 포지셔닝

Astro Studio는 단일 모델의 데모 셸이 아니며, 일회성 이미지 출력을 강조하는 웹 도구도 아닙니다.

창작자가 소유하는 이미지 생성 워크스테이션에 가깝습니다：

- 서드파티 이미지 생성 API를 자유롭게 구성할 수 있습니다
- 다양한 프로바인더를 하나의 데스크톱 진입점에 연결할 수 있습니다
- 같은 공간에서 생성, 조회, 필터링, 즐겨찾기, 편집 이어하기를 할 수 있습니다
- 제작 히스토리를 장기적으로 관리 가능한 소재 자산으로 만들 수 있습니다

워크플로우가 특정 플랫폼에 묶이지 않도록 하고 싶은 분들을 위해 설계되었습니다.

## 이 제품을 만든 이유

현재 많은 AI 이미지 제품들이 모델의 능력에 초점을 맞추고 있지만, "제작 이후의 처리"에는 거의 관심을 기울이지 않습니다：

- 히스토리가 금방 어지러워집니다
- 마음에 드는 결과를 정리하기 어렵습니다
- 계속 반복하고 싶을 때 컨텍스트가 끊어집니다
- 프로바인더를 바꾸면 사용 습관도 모두 새로 시작해야 합니다

Astro Studio는 이 흩어진 단계들을 재구성하여, 이미지 생성을 일련의 시험에서 연속적인 창작 프로세스로 바꾸고자 합니다.

## 핵심 가치

### 자유로운 연결

Astro Studio의 핵심은 특정 플랫폼을 전제로 하는 것이 아니라, API Key, Base URL, 모델 엔드포인트를 자유롭게 설정하여 자신의 방식으로 서드파티 이미지 생성 능력을 연결할 수 있도록 하는 것입니다.

### 통합 진입점

공식 서비스, 프록시 게이트웨이, 호환 가능한 서드파티 프로바인더를 사용하든, 여러 웹 관리 화면을 오가는 것이 아니라 하나의 데스크톱 인터페이스 내에서 작업을 완료하는 것을 목표로 합니다.

### 장기적인 창작을 위해

"생성의 성공"뿐만 아니라, 히스토리 축적, 작품 회고, 즐겨찾기 정리, 편집의 지속, 이후의 반복에도 주목하여, 생성하는 순간 사라지는 것이 아니라 창작 결과를 남깁니다.

### 데스크톱 경험

웹 도구와 비교하여, Astro Studio는 안정성, 집중력, 지속적인 사용감을 강조합니다. 가끔 방문하는 웹페이지가 아니라, 매일 열어보는 창작 워크스테이션을 지향합니다.

## 대상 사용자

- AI로 이미지를 자주 생성하는 디자이너, 일러스트레이터, 콘텐츠 크리에이터
- 여러 이미지 생성 서비스를 동시에 사용하는 개인 사용자 또는 소규모 팀
- API 프로바인더를 자유롭게 전환하고 싶고, 단일 플랫폼에 묶이지 않으려는 분
- 히스토리, 즐겨찾기, 소재, 설정을 장기간 데스크톱 환경에서 관리하고 싶은 분

## 할 수 있는 것

- 프롬프트를 입력하여 빠르게 이미지 생성 시작
- 자체 서드파티 이미지 생성 API 연결
- 다양한 서비스와 모델 간의 통합된 사용 진입점 구축
- 전체 히스토리를 탐색하고, 과거 제작 결과를 회고
- 마음에 드는 이미지를 즐겨찾기에 등록하고 폴더로 정리
- 기존 이미지를 기반으로 편집과 반복 계속하기
- 검색으로 과거에 생성한 콘텐츠를 빠르게 찾기
- 더 안정적인 로컬 데스크톱 방식으로 창작 프로세스 전체 관리

## 우리가 믿는 것

Astro Studio는 이미지 제작 도구의 미래가 단일 모델의 부속 인터페이스가 아니라, 더 중립적이고, 오픈하며, 장기적으로 사용할 수 있는 플랫폼이어야 한다고 믿습니다.

모델은 변하고, 프로바인더는 변하고, 인터페이스도 변합니다. 하지만 에코시스템이 변할 때마다 크리에이터가 워크플로우의 통제를 잃어서는 안 됩니다.

그래서 Astro Studio는 "선택권"을 사용자에게 돌려주고자 합니다：

- 누구와 연결할지 선택
- 어떻게 사용할지 선택
- 어떻게 창작 히스토리를 보존할지 선택
- 어떻게 소재 세계를 구성할지 선택

## 로컬 빌드

소스 코드에서 Astro Studio를 로컬로 실행하려면, 먼저 기본 환경을 준비하세요：

- Node.js `22+`
- npm `11+`
- Rust stable toolchain
- 사용 중인 OS에 필요한 Tauri 의존성

프로젝트 의존성 설치：

```bash
npm install
```

데스크톱 앱 개발 모드로 시작：

```bash
npm run tauri dev
```

프론트엔드만 시작하려면：

```bash
npm run dev
```

프로덕션 빌드 실행：

```bash
npm run build
npm run tauri build
```

현재 GitHub 자동 릴리스 프로세스는 Windows 인스톨러만 생성합니다. 로컬에서 macOS 버전을 빌드하여 개인적으로 사용하는 경우, 앱은 일반적으로 사용자의 머신에서 작동하지만, 공개 배포에는 Apple 서명과 공증이 필요합니다.
