Astro Studio は AI 画像作成のために設計されたデスクトッププラットフォームです。特定のモデルプロバイダーに縛られるのではなく、サードパーティの画像生成 API を自由に接続し、異なるサービスやモデル、継続的な創作活動を一箇所から管理できるようにします。

単なる一回の生成リクエストを解決するだけでなく、画像生成を安定して使い続けられる長期的な創作ワークフローに変えることを目指しています。

## 製品の位置づけ

Astro Studio は単一モデルのデモシェルではなく、一度きりの画像出力を重視するウェブツールでもありません。

より正確に言えば、創作者自身が所有する画像生成ワークステーションです：

- サードパーティの画像生成 API を自由に設定できる
- 異なるプロバイダーを一つのデスクトップエントリに接続できる
- 同じ空間で生成、閲覧、フィルタリング、お気に入り登録、編集の続きができる
- 作成履歴を長期的に管理できる素材資産にできる

ワークフローを特定のプラットフォームに縛られたくない方のために設計されています。

## この製品を作った理由

現在多くの AI 画像製品はモデルの能力に重点を置いていますが、「作成後の処理」にはあまり注意を払っていません：

- 履歴がすぐに乱れる
- 気に入った結果が整理しにくい
- 続きを作りたいとき、コンテキストが途切れている
- プロバイダーを変えると、使用習慣もすべてやり直し

Astro Studio はこれらのばらばらなステップを再構成し、画像生成を一つの試行から連続的な創作プロセスに変えたいと考えています。

## コアバリュー

### 自由な接続

Astro Studio の核は特定のプラットフォームを前提とすることではなく、API Key、Base URL、モデルエンドポイントを自由に設定でき、自分の方法でサードパーティの画像生成能力を接続できるようにすることです。

### 統一エントリ

公式サービス、プロキシゲートウェイ、互換性のあるサードパーティプロバイダーのいずれを使っても、複数のウェブ管理画面を行き来するのではなく、一つのデスクトップインターフェース内で操作を完結させることを目指しています。

### 長期的な創作のために

「生成の成功」だけでなく、履歴の蓄積、作品の振り返り、お気に入り整理、編集の継続、その後の反復にも注目し、生成した瞬間に消えてしまうのではなく、創作成果を残します。

### デスクトップ体験

ウェブツールと比較して、Astro Studio は安定性、集中力、継続的な使用感を重視します。たまに訪れるウェブページではなく、毎日開く創作ワークステーションを目指しています。

## 対象ユーザー

- AI で画像を頻繁に生成するデザイナー、イラストレーター、コンテンツクリエイター
- 複数の画像生成サービスを同時に使用する個人ユーザーまたは小規模チーム
- API プロバイダーを自由に切り替えたい、単一プラットフォームに縛られたくない方
- 履歴、お気に入り、素材、設定を長期間デスクトップ環境で管理したい方

## できること

- プロンプトを入力して素早く画像生成を開始
- 独自のサードパーティ画像生成 API を接続
- 異なるサービスやモデル間で統一された使用エントリを構築
- 完全な履歴を閲覧し、過去の作成結果を振り返る
- 気に入った画像をお気に入りに登録し、フォルダで整理
- 既存の画像をベースに編集と反復を続ける
- 検索で過去に生成したコンテンツを素早く見つける
- より安定したローカルデスクトップ方式で創作プロセス全体を管理

## 私たちが信じるもの

Astro Studio は、画像作成ツールの未来は単一モデルの付属インターフェースであるべきではなく、より中立で、オープンで、長期的に使えるプラットフォームであるべきだと信じています。

モデルは変わり、プロバイダーは変わり、インターフェースも変わります。しかし、エコシステムが変化するたびにクリエイターがワークフローの制御を失うべきではありません。

だからこそ Astro Studio は「選択権」をユーザーに返したいと考えています：

- 誰と接続するか選ぶ
- どう使うか選ぶ
- どう創作履歴を保存するか選ぶ
- どう素材世界を構成するか選ぶ

## 入手方法

[Releases](https://github.com/3inchtime/astro_studio/releases) からお使いのプラットフォームのインストーラーをダウンロードしてください。

初回起動後に API 設定を完了するだけで、Astro Studio をサードパーティ画像生成サービスの統一デスクトップエントリとして使用できます。

## ローカルビルド

ソースコードから Astro Studio をローカルで実行する場合は、まず基本環境を準備してください：

- Node.js `22+`
- npm `11+`
- Rust stable toolchain
- お使いの OS に必要な Tauri の依存関係

プロジェクトの依存関係をインストール：

```bash
npm install
```

デスクトップアプリの開発モードで起動：

```bash
npm run tauri dev
```

フロントエンドだけを起動する場合：

```bash
npm run dev
```

プロダクションビルドを実行：

```bash
npm run build
npm run tauri build
```

現在の GitHub 自動リリースフローは Windows インストーラーのみを生成します。ローカルで macOS 版をビルドして個人的に使用する場合、アプリは通常お使いのマシンで動作しますが、公開配布には Apple の署名と公証が必要です。
