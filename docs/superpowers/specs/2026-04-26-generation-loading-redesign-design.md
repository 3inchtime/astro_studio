# 生成中 Loading 动画重设计设计稿

Date: 2026-04-26

## 概述

重做图片生成中的 loading 区域，完全替换当前 `DreamBubbles` 泡泡动画。新的设计方向为「星环仪式感动画 + 电影感滚动文案」：让等待过程更像作品正在显影，而不是程序在空转。

本次改动仅作用于生成页对话流中，`message.status === "processing"` 时的助手消息气泡。完成态、失败态、图片展示逻辑保持不变。

## 目标

- 提升等待过程的期待值，让生成中的状态更有情绪和品牌识别度
- 用柔和光影、缓慢呼吸、低频扫光来传达“作品正在孕育”的感觉
- 增加滚动文字系统，循环展示电影感状态文案，而不是单一“生成中...”
- 确保亮色模式和暗色模式都成立，不能只在深色背景下好看
- 保持克制，不要喧宾夺主，不影响提示词对话内容的阅读

## 视觉方向

### 核心气质

- 主方向：高级梦境
- 氛围关键词：星云、雾光、显影、降临、柔和、精致
- 情绪目标：让用户感觉画面正在被调色、聚焦、显现

### 最终组合

- 动画骨架采用「星环剧场」方向
- 文字系统采用「星核显影」方向
- 结果是一套有仪式感但不浮夸、有电影感但不过度戏剧化的生成中场景

## 组件结构

在 [MessageBubble.tsx](/Users/chen/Workspace/astro_studio/src/components/generate/MessageBubble.tsx) 中，`processing` 分支替换为新的 `GenerationLoadingScene`（名称可调整，但需独立成可读组件），结构分为四层：

1. 外层消息卡片
2. 星云场景层
3. 文案信息层
4. 进度暗示层

### 1. 外层消息卡片

- 保持现有消息气泡位置和圆角体系，继续作为助手消息的一部分出现
- 卡片本体从普通 `bg-surface` 升级为带主题感的柔和渐变面板，但仍服从现有设计系统的边框和阴影语言
- 需要保留进入和退出动画，避免和完成态图片切换时产生突兀断层

### 2. 星云场景层

场景中心为一个“正在显影”的光核，外部包裹 2 到 3 层呼吸扩散的星环，配合柔和雾光与低频扫光。

构成元素：

- `core`：中央发光核心，负责焦点感
- `rings`：2 到 3 层透明星环，做缓慢扩散或轻微缩放呼吸
- `mist`：背景雾状渐变，提供高级梦境氛围
- `beam`：低频扫光，横向或斜向掠过一次，作为“显影中”的提示
- `dust`：少量星尘高光点，数量少、亮度低，只做静态点缀或极轻微漂移

约束：

- 不出现尖锐旋转、快速跳动、强闪烁
- 节奏以缓慢、层叠、连续为主
- 动画不模拟真实进度条，只营造“正在处理”的可信等待感

### 3. 文案信息层

文案层放在场景下方，与动画形成一个完整的生成状态卡片。

包含：

- 主标题：单句核心状态文案
- 副文案：1 句简短解释，用于补充“正在显影”的氛围
- 滚动字幕：横向循环的电影感短句列表

推荐默认文案方向：

- 主标题：`画面正在穿过雾光`
- 副文案：强调光影聚拢、层次校准、作品即将出现
- 滚动字幕示例：
  - `宇宙正在为你调色`
  - `画面正在穿过雾光`
  - `星云正在校准层次`
  - `最终镜头正在靠近`

规则：

- 文案长度要适合中英文双语展示，避免超长句影响滚动节奏
- 滚动文字应连续、柔和、可循环，不使用突兀跳切
- 字幕容器两侧用渐隐遮罩，形成“从雾中穿过”的感觉
- 字幕整体允许轻微呼吸透明度变化，但不能频繁闪烁

### 4. 进度暗示层

底部保留一条非精确进度性质的光带或脉冲条，用于补充“系统仍在工作”的反馈。

规则：

- 不显示具体百分比
- 不与真实生成进度绑定
- 视觉上更像能量流或光谱推进，而不是传统 loading bar

## 动效设计

### 节奏

- 总体节奏慢，推荐主循环在 `4s` 到 `7s` 区间
- 各动画错峰执行，避免所有元素同频呼吸导致机械感
- 切换到完成态图片时，loading 先柔和淡出，再由图片浮现

### 建议动效拆分

- 中央光核：轻微缩放 + 亮度呼吸
- 第一层星环：低幅缩放呼吸
- 第二层星环：更慢的透明度浮动
- 扫光：低频平移或斜向扫过
- 雾层：极慢背景位移或透明度变化
- 滚动字幕：匀速横向 marquee 效果
- 进度光带：缓慢位移或波纹推进

### 减少动态偏好

必须支持 `prefers-reduced-motion: reduce`：

- 停用连续平移动画
- 保留静态层次、光影和清晰文案
- 只允许非常轻微或直接禁用的透明度变化

## 亮色 / 暗色模式策略

这是本次设计的强约束，不是附加优化。

### 暗色模式

- 保持深色星云感，允许更明显的紫蓝雾光与中心辉光
- 通过半透明边框和柔和内发光提升层次
- 保证文案对比度足够，尤其是副文案和滚动字幕不能发灰难读

### 亮色模式

- 不直接照搬暗色配色，而是转成“晨雾显影”质感
- 背景应使用浅暖底色上的淡蓝紫、淡粉雾光，而不是纯白生硬底
- 星环边框和雾光需要收敛亮度，防止在浅底上泛脏或发灰
- 文案颜色需更接近主题前景色，避免因为透明度过高而失去清晰度

### 实现约束

- 颜色优先依赖 [globals.css](/Users/chen/Workspace/astro_studio/src/styles/globals.css) 中的主题变量扩展，不直接把深色 RGBA 常量写死在组件里
- 场景需要同时定义 light / dark 两套渐变、边框、阴影、发光变量
- 如果需要新增 loading 专用 CSS 变量，应挂在全局主题系统内，而不是内联散落在 JSX 中

## 国际化策略

- 中英文都要提供主标题、副文案和滚动字幕内容
- 新文案写入 [zh-CN.json](/Users/chen/Workspace/astro_studio/src/locales/zh-CN.json) 与 [en.json](/Users/chen/Workspace/astro_studio/src/locales/en.json)
- 不把多条滚动字幕硬编码在组件内部
- 文案 key 建议按 `generate.loading.*` 组织，例如：
  - `generate.loading.title`
  - `generate.loading.subtitle`
  - `generate.loading.ticker1`
  - `generate.loading.ticker2`

## 实现原则

- 新 loading 组件应独立，避免继续把复杂动画塞在 `MessageBubble.tsx` 内
- 样式尽量集中在全局样式系统，避免大量内联 style 难以维护
- 保持现有 `framer-motion` 使用方式一致，只在必要的进入/退出过渡处使用 motion
- 不引入新依赖

## 验收标准

满足以下条件才算完成：

- 生成中的旧泡泡动画被完全替换
- 新 loading 在亮色模式和暗色模式下都具有完整层次和良好可读性
- 主标题、副文案、滚动字幕均正常显示，且支持中英切换
- 滚动字幕在窄宽度下不会出现明显裁切 bug 或突兀换行
- loading 与完成态图片的过渡自然，没有强烈闪烁或布局抖动
- `prefers-reduced-motion` 下能正常降级
- 没有影响失败态、完成态、收藏等其他消息气泡功能

## 范围外

- 不接入真实生成进度百分比
- 不修改图片完成后的展示布局
- 不改变对话页整体结构
- 不新增声音、粒子爆炸、3D 旋转等强表现效果
- 不为本次 loading 单独引入 canvas、svg 动画引擎或 Lottie
